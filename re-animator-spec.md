## 0. 설계 원칙 (레퍼런스 영상 정량 분석 결론)

스타일리시한 카메라워크의 실체는 생성 모델이 아니라 편집 레이어다. 따라서:
- 카메라워크는 원칙적으로 I2V에 맡기지 않는다. 워커서버가 후처리로 얹는다. (유일 예외: orbit, 2절 참조)
- orbit을 제외한 모든 I2V 프롬프트에 "static camera, fixed shot, no camera movement"를 강제 삽입. 네거티브: warping, morphing, character deformation.
- I2V에는 인물 국소 모션만 기대한다(입, 표정, 머리카락, 옷자락). 핑퐁 루프는 사용 금지.

## 1. 오버스캔과 업스케일 (재생성 금지 원칙)

- 이미지 생성은 기존대로 gpt-image-2 1536x1024. 16:9 크롭 시 1536x864 작업 캔버스 = 720p 출력 대비 120% 오버스캔이 자동 확보된다. 별도 대형 생성 불필요.
- 씬당 줌 총량(zoom_rate x duration) 기본 상한 20%. 초과 씬과 1080p 출력 시에만 워커에서 애니메 특화 업스케일러(Real-ESRGAN 계열) 2배 패스를 적용한다.
- G2 검수를 통과한 이미지의 재생성은 오버스캔 목적으로 절대 수행하지 않는다. 부족분은 항상 업스케일로 해결한다(검수 무효화 방지).

## 2. 카메라워크: 3계층 아키텍처 + 키프레임 테이블 단일 소스

카메라워크는 구현 경로에 따라 3계층으로 나뉜다.

계층 A — 단일 레이어 crop (대부분의 프리셋)
- 프리셋: push_in(1.0-4.0%/s, 감정 피크 6-10%/s, 항상 소량 대각 드리프트 동반), pull_out, pan, static, shake, crash_zoom, whip.
- crash_zoom은 카메라 이동이 아니라 동일 피사체의 와이드/바스트/익스트림클로즈업 3장을 각 0.4-0.8초 하드컷한 컷 클러스터다. 재생성 단계에 "리프레이밍 3프레이밍 세트" 잡 타입을 추가하라.
- whip은 씬 전환 속성: 12프레임 내외 모션블러 스와이프.

계층 B — 2레이어 스케일 (달리줌 계열, 기존 스펙의 레이어 분리 활용: 인물 마스크 고정 + 배경만 스케일)
- parallax_push: 배경을 인물 대비 초당 1-2%p 더 밀어 깊이감 생성. 빈도 제한 없음. talk/idle 씬의 디폴트 후보로 승격 — 일반 push_in보다 입체감이 좋다.
- vertigo: 배경을 초당 5%p 이상 역방향 스케일. 각성·배신·진실 자각 등 shock 태그 씬 전용, 에피소드당 1-2회 권장. VLM이 해당 태그 씬에만 제안.

계층 C — I2V 위임 (유일 예외)
- orbit: 시점 회전은 2D 후처리로 불가능하므로 이 프리셋만 I2V 프롬프트에 카메라 지시("slow orbit around subject")를 넣는다. 사용 최소화 — 감정 정점·등장 씬 등 에피소드당 소수. orbit 클립 위에는 후처리 카메라워크를 겹치지 않는다(이중 무빙 방지). 클라이언트 프리뷰 불가이므로 이 프리셋만 "프록시 렌더 필수" 라벨을 붙인다.

공통 구현 규칙
- camera_work 스키마: { preset, zoom_rate_pct_per_s, drift_px_per_s{x,y}, bg_scale_delta_pct_per_s(계층 B용), easing, duration_s, shake_seed }
- 수식 모듈은 파라미터로부터 프레임별 키프레임 테이블(JSON)을 생성하는 순수 함수 하나로 구현한다. 계층 A는 단일 crop 트랙, 계층 B는 background/character 2트랙. 워커(ffmpeg 프레임별 crop·스케일 합성, zoompan 수식 직접 기술 금지)와 웹앱(Web Animations API 키프레임 주입, 계층 B는 레이어별 주입) 양쪽은 이 테이블만 소비한다. 수식을 두 벌 구현하지 마라.
- 셰이크는 shake_seed 기반 시드 PRNG로 양쪽 동일 궤적 재현.
- 골든 테스트: 대표 파라미터 세트에 대해 t=0/25/50/75/100% 시점의 좌표(계층 B는 레이어별)가 워커·웹앱 간 2px 이내 일치해야 통과.
- 컷 리듬 자동 제안 목표: 숏 길이 중앙값 3.5-4초, 2초 미만 초단컷 25-30%, 8초 이상은 감정 정점에 소수. action/shock 태그 씬에는 초단컷 3연속 클러스터를 제안.

## 3. 모션 티어 (VLM 자동 분류 — 추가 API 호출 없음)

기존 VLM 장면 서술 호출의 출력 스키마에 다음을 추가한다:
  motion_tier: talk | idle | emote | action
  tier_confidence: 0-1
  tier_evidence: 근거 한 줄
  motion_prompt_hint: 티어 규칙을 준수한 I2V 모션 서술
- VLM 입력에 현재 컷 + 앞뒤 컷 썸네일을 함께 제공한다.
- 판단 신호를 VLM 프롬프트에 명시: 말풍선=talk, 스피드라인·집중선·의성어 글자=action, 클로즈업+과장 표정(땀방울 등)=emote, 정경·사물·뒷모습=idle, 인접 컷 간 동일 인물의 자세 차이 큼=action+보간 후보.
- 말풍선과 action 판단이 충돌하면 talk 우선하되 motion_prompt_hint에 동작 요소를 허용하는 talk+action 하이브리드로 처리.
- tier_confidence 낮은 씬은 기존 "미결만 보기" 필터에 포함.
- 티어별 I2V 요청: talk 3-4초(입 움직임+미세 표정, 몸 고정 / 정밀 립싱크는 v2, v1은 립플랩), idle 2-3초(숨·머리카락·옷자락), emote 2초(표정 A→B 전환만), action 1-2초 강제.

## 4. 액션 씬

- 디폴트 경로: 단일 이미지 + 강한 모션 프롬프트 + 1-2초 강제.
- 동작 보간(옵트인): G2의 기존 병합(merge) 기능 확장. 연속 두 컷을 병합한 씬은 이미지 2장을 보유하며 "동작 보간" 토글이 나타나고, 켜면 I2V에 first/last frame으로 전달. VLM이 감지한 보간 후보는 G2에 배지로 표시. 새 UI 패턴을 만들지 말 것.
- 실패·아티팩트 폴백: 정지화 + 강한 카메라워크 + 이펙트 오버레이(플래시 프레임 1-2f, 스피드라인 PNG 시퀀스, ±4-10px 시드 셰이크). 리롤 강요 금지.

## 5. 씬 길이: 2단계 duration (TTS·I2V 병렬 유지)

- duration_est: 씬 확정 시 대사 글자 수 기반 어림값. 다국어 규칙은 11절을 따른다(선택 언어들의 최댓값 기준). 카메라워크 초안·프리뷰·비용 추정에 사용.
- I2V 요청 길이 = est x 1.15 마진(엔진 스텝 올림). TTS와 I2V는 병렬 실행 — 선행 의존성을 만들지 마라.
- duration_final: TTS 도착 시 확정(언어별, 11절). final < 클립이면 트림(대부분), final > 클립이면 마지막 프레임 홀드 + 카메라워크 지속. 카메라워크 키프레임 테이블은 final 확정 시 재계산(순수 연산).
- 길이 부족 보정: idle/emote는 0.6-0.8배 슬로우 + 프레임 보간(RIFE 또는 minterpolate)으로 24fps 유지 허용. talk는 슬로우 금지. 루프는 전 티어 금지.

## 6. 오디오 채움 (무음 구간 최소화)

기존 VLM 호출 스키마에 추가:
  audio_suggestions: [{ type: sfx|vocal_reaction|insert_line, text, speaker(배역ID), timing: start|mid|end, confidence }]
- 판단 근거: 의성어 글자가 그려진 컷은 그 소리를 SFX로, 놀람·한숨 표정은 해당 배역 리액션 발성으로, 인물이 있는데 말풍선 없는 컷은 짧은 삽입 대사 후보로.
- 생성 경로: sfx는 ①Grok I2V 네이티브 오디오 우선 ②SFX 라이브러리 매핑 테이블. vocal_reaction과 insert_line은 해당 배역에 바인딩된 TTS 보이스로 생성(목소리 일관성 유지).
- 검수: 세 종류 모두 기본 승인. insert_line은 원작에 없는 창작이므로 가시성으로 관리한다 — 펼친 화면 프리뷰에 원클릭 on/off와 트랙별 솔로/뮤트, 목록 상단에 "삽입 대사 일괄 끄기" 버튼.
- 합성: 오디오 3트랙(BGM / 대사·발성 / SFX). 대사·발성 구간에서 BGM -6dB 덕킹.

## 7. 자막

- 하단 중앙, 텍스트 하단 = 화면 높이 92-95% 지점, 블록 높이 4-5%(720p 기준 약 30px), 흰색 + 검은 외곽선 2px, 배경 박스 없음.
- 타이밍은 해당 오디오에 종속: 시작 0.1초 전 표시, 종료 0.2초 후 소멸. ASS로 생성해 ffmpeg subtitles 필터로 번인. 삽입 대사도 동일 규칙. 자막 텍스트 언어는 11절을 따른다.

## 8. 웹앱 프리뷰 (2단)

① 클라이언트 시뮬레이션(기본, 무비용): 씬 정지 이미지 위에 키프레임 테이블을 Web Animations API로 재생. 계층 B는 인물/배경 레이어를 각각 재생. 씬 아코디언 펼침 화면에 플레이어 + 파라미터 슬라이더(zoom, drift, bg_scale_delta, duration), 조정 즉시 반영, 저장은 camera_work JSON만 갱신(렌더 잡 생성 안 함). crash_zoom은 3프레이밍 이미지 타이밍 스위칭, whip은 CSS 근사. 근사 렌더임을 UI에 "근사" 라벨로 정직하게 표시. orbit은 클라이언트 프리뷰 불가 — "프록시 렌더 필수" 라벨.
② 워커 프록시 렌더(요청 시): "정확 미리보기" 버튼 → 480p 프록시 렌더 잡(우선순위 리롤 동급) → 완료 시 플레이어 교체. camera_work 해시로 캐시, 변경 시 무효화.

## 9. 동영상 편집 레이어 UI (씬 목록 재정의)

원칙: 카메라워크·전환·이펙트·오디오 제안 등 연출 설정은 전부 자동으로 채워지며, 사람은 예외만 만진다. 이를 위해 접힌 줄을 대폭 단순화한다.

접힌 씬 줄 — 다음 4개 요소만 표시한다:
  ① 대사 텍스트 (작업 언어 주 표기 + 한국어 보조 표기, 길면 말줄임 — 11절)
  ② 씬 길이 (duration_est 초, TTS 확정 후 작업 언어의 duration_final로 교체 표시)
  ③ 발화자 (배역 이름. 무대사 씬은 "-")
  ④ 모션 티어 — 드롭다운(talk/idle/emote/action). 접힌 상태에서 바로 변경 가능해야 하며, 변경 시 티어별 I2V 요청 규칙이 즉시 재적용된다.
- 기존 접힌 줄의 썸네일·상태뱃지·설정 아이콘 4종·누적 비용·삽입 대사 아이콘은 접힌 줄에서 제거한다.
- 카메라워크 프리셋·파라미터, 전환, 오디오 제안 on/off, 상태·비용 정보는 펼친 화면에서만 보이고 편집한다.
- "삽입 대사 일괄 끄기" 버튼과 "미결만 보기" 필터는 목록 상단에 유지한다.
- 아코디언 규칙(항상 최대 1개 펼침, 승인=승인하고 다음 씬 열기)은 기존 스펙 그대로.

## 10. 다국어 파이프라인 (복수 타깃 언어, 언어별 버전 출력)

프로젝트 설정에서 번역 대상 언어를 복수 선택할 수 있게 한다(현재 요구: 일본어 + 영어. 이후 추가 가능). 언어 추가 시 해당 언어의 번역·TTS 잡만 증분 생성한다.

자산 분리 원칙
- 언어 무관 자산(공유, 1벌): 재생성 이미지, I2V 클립, 카메라워크 키프레임 테이블, 씬 구조·전환, SFX·BGM. 립싱크가 아닌 립플랩이므로 입 모양은 언어에 묶이지 않는다 — 이 공유가 이 설계의 핵심 비용 이점이다.
- 언어별 자산(타깃 언어 수만큼): 번역 텍스트, TTS 오디오(대사·리액션 발성·삽입 대사), 자막 ASS, duration_final, 트림/홀드 지점.

데이터 모델
  dialogue: { source_text, ko_text, speaker, tracks: { ja: {text, tts_audio, duration_final, status}, en: {...} } }
- VLM 읽기 단계에서 선택된 전체 언어를 한 번의 호출로 동시 번역한다(언어당 재호출 금지). 원어가 한국어인 경우 source_text = ko_text로 두고 표시를 중복하지 않는다.
- 배역 보이스 바인딩은 언어 공통 1개를 기본으로 하되(멀티링구얼 보이스 전제), 언어별 오버라이드 슬롯을 둔다(특정 언어에서 발음이 어색한 경우).

duration 처리 (5절 보정)
- duration_est = 선택된 언어들의 글자 수 어림값 중 최댓값(언어별 발화 속도 상수를 언어 설정에 포함: 한국어 4-5자/s, 일본어 6-7모라/s, 영어 14-16자/s 수준에서 시작해 실측으로 보정). I2V 요청 길이 = 이 최댓값 x 1.15 — 한 클립이 모든 언어 버전을 수용하게 한다.
- duration_final은 언어별로 확정. 트림/홀드와 카메라워크 키프레임 재계산도 언어별로 수행한다(순수 연산이므로 무비용). 즉 일본어판과 영어판은 씬 길이가 미세하게 다른 별개의 타임라인이다.

UI
- 화면 상단에 "작업 언어" 전환 토글(ja/en)을 둔다. 목록·펼침 화면의 대사 표시, 프리뷰의 오디오·자막, duration 표시가 토글된 언어 기준으로 일괄 전환된다. 한국어는 어느 언어에서든 보조 표기(작은 글씨)로 항상 병기한다.
- G1(컷 읽기·검수) 셀은 원어 / 한국어 / 선택 언어 전체(ja, en)를 모두 표시하고 인라인 수정 가능. 원어 수정 시 한국어·번역 언어 재번역을 제안(자동 덮어쓰기 금지, 사람이 승인). 이후 단계는 작업 언어 + 한국어 병기 2줄.
- 언어별 진행 상태(번역 완료/TTS 완료/미검수)를 목록 상단에 언어 탭 뱃지로 요약 표시. "미결만 보기" 필터는 작업 언어 기준으로 동작.

출력
- 자막 번인과 TTS는 작업 언어 텍스트 기준. 한국어는 작업자 이해용 병기이며 산출물에는 포함하지 않는다. 삽입 대사·리액션 발성 제안(6절)도 각 언어로 생성하고 한국어 병기를 함께 저장한다.
- 최종 합성은 언어별 별도 렌더 잡: 공유 비주얼 트랙 + 해당 언어의 오디오·자막·타임라인을 결합해 파일명에 언어 코드를 붙여 출력한다(예: ep01_ja.mp4, ep01_en.mp4).
- 비용 대시보드는 공유 자산 비용과 언어별 증분 비용(TTS·번역)을 분리 표기한다.

## 11. 구현 순서

1) 카메라워크 수식 모듈(계층 A/B 트랙 포함) + 키프레임 테이블 + 골든 테스트  ← **[완료 · 9123986]** (골든 103 pass, 워커↔웹 2px)
2) 워커 렌더러(프레임별 crop·2레이어 합성, 업스케일 트리거 포함)  ← **[계층 A 완료]** (sendcmd crop, 통합 11 pass; 계층 B/ESRGAN 후속)
3) 웹앱 클라이언트 프리뷰 + 파라미터 편집 + 씬 목록 재정의(9절)  ← **[완료]** (프리뷰·편집 + §9 아코디언: 접힌 줄 4요소·미결만 보기·삽입대사 일괄끄기)
4) VLM 스키마 확장(모션 티어 + 오디오 제안 + 다국어 동시 번역 + vertigo/보간 후보 태그)  ← **[분류 산출 완료]** (classify.mjs strict 스키마+prompts.json: motion_tier·audio_suggestions·보간후보; 다국어 동시번역은 Phase 5)
5) 다국어 데이터 모델 + 작업 언어 토글 + 언어별 duration/타임라인  ← **[데이터모델·번역생성·G1 표시 완료 / 작업언어 토글·언어별 TTS·compose 후속]** (DialogueBubble.tracks 하위호환, Project.targetLanguages, translateScenesMultilang 한 콜 동시번역, extract 조건부 배선, 대사 행에 언어별 번역 표시·인라인 수정)
6) 티어별 I2V 요청 규칙 + duration 2단계 + 트림/홀드/슬로우  ← **[티어→I2V 완료]** (estimateVideoSeconds 티어 길이범위, buildVideoPrompt action 절제완화·motionPromptHint; duration final/트림/홀드/슬로우는 TTS 도착 의존 → Phase 8)
7) crash_zoom 3프레이밍 잡 + 병합 확장 동작 보간 + orbit I2V 예외 경로
8) 언어별 TTS·자막 ASS + 오디오 3트랙 합성 + 덕킹 + whip·이펙트 오버레이 + 프록시 렌더 + 언어별 출력 잡

각 단계는 실행 가능한 상태로 커밋. 완료마다 spec v0.2에 반영.
